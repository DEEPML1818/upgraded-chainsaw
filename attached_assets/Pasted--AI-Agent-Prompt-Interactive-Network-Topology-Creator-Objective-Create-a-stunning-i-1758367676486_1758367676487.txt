## üé® **Design Requirements**

### **Visual Style Standards:**
- **Modern Gradient Background**: Purple-blue gradient (`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`)
- **Glass Morphism**: White containers with `backdrop-filter: blur(10px)` and transparency
- **Animated Title**: Large, glowing title with `titleGlow` animation
- **Rounded Corners**: 15-20px border radius on all containers
- **Shadow Effects**: `box-shadow: 0 20px 60px rgba(0,0,0,0.2)` on main container

### **Component Styling:**
- **Network Zones**: Color-coded zones with dashed borders and gradient backgrounds
- **Network Components**: White rounded rectangles with colored borders and icons
- **Status Indicators**: Green pulsing dots on each component
- **Hover Effects**: Components lift up with `transform: translateY(-8px) scale(1.05)`
- **Icon Animation**: Bouncing icons with `iconBounce` keyframe

### **Connection System:**
- **SVG Paths**: Curved connections using `<path>` elements with quadratic curves
- **Connection Types**: 4 distinct line styles:
  - üîµ **Standard**: Solid blue lines (`#2196f3`)
  - üü¢ **Secure**: Solid green lines (`#4caf50`) 
  - üü† **Encrypted**: Dashed orange lines (`#ff9800`)
  - üî¥ **High Traffic**: Thick red lines (`#f44336`)
- **Mid-Point Labels**: Small labeled badges showing protocol/function at connection midpoints
- **Flow Animation**: Animated stroke-dasharray for data flow visualization

## üìê **Responsive Architecture**

### **Viewport-Based Sizing:**
```css
- Container: 95vw √ó 90vh (full viewport scaling)
- Zones: Position using vh/vw units (e.g., top: 16vh, left: 2vw)
- Components: Size using vw/vh (e.g., width: 6vw, height: 8vh)
- SVG: viewBox="0 0 100 100" with preserveAspectRatio="none"
- Connections: Percentage coordinates (0-100) with vector-effect="non-scaling-stroke"
```

### **Media Queries:**
- **Large (1400px+)**: Full detailed view
- **Medium (1000-1400px)**: Proportional scaling
- **Small (800-1000px)**: Compact layout
- **Mobile (<800px)**: Minimum viable sizes

## üîß **Technical Implementation**

### **HTML Structure:**
```html
<body>
  <div class="title">[Architecture Name] Network Topology</div>
  <div class="topology-container">
    <!-- Network Zones -->
    <div class="network-zone [zone-type]">
      <div class="zone-label">[Zone Name]</div>
    </div>
    
    <!-- Network Components -->
    <div class="network-component [component-type]">
      <div class="icon">[Emoji]</div>
      <div>[Component Name]</div>
      <div class="status-indicator"></div>
    </div>
    
    <!-- SVG Connections -->
    <svg class="connection" width="100%" height="100%" viewBox="0 0 100 100">
      <path class="connection-line [connection-type]" d="M x1,y1 Q mx,my x2,y2" />
    </svg>
    
    <!-- Labels -->
    <div class="connection-mid-label [label-type]">[PROTOCOL]</div>
    
    <!-- Legend -->
    <div class="legend">...</div>
  </div>
</body>
```

### **CSS Animation Requirements:**
- **titleGlow**: Text shadow animation (3s infinite alternate)
- **iconBounce**: Component icon bouncing (2s infinite)
- **statusPulse**: Status indicator pulsing (2s infinite)
- **dataFlow**: Connection line animation (3s infinite)
- **labelFloat**: Label floating animation (3s infinite)
- **particleFloat**: Background particles (8s infinite)

### **JavaScript Functionality:**
- No localStorage/sessionStorage (not supported)
- Use CSS animations only
- Ensure all interactions are CSS-based
